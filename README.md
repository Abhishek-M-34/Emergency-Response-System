# 🏨 LOOP ERS – Hotel Emergency Response System

A cloud‑powered, AI‑assisted emergency management platform designed for hotels.
Guests can instantly report medical, fire, security, or natural disaster emergencies.
Emergency Control and Staff can manage alerts in real time across all devices.

**🚀 Live demo:** [https://hotel-ers.web.app](https://hotel-ers.web.app)

---

## 🔥 Key Features

### 👤 Guest
- Report emergencies with one click (Medical / Fire / Security / Natural Disaster)
- Select severity (Normal / Medium / Serious) or let it auto‑select after 5s
- **AI Safety Guide** – get a step‑by‑step action plan generated by Gemini AI
- View personal emergency history
- Receive broadcast warnings from Emergency Control

### 🎛️ Emergency Control (EC)
- Real‑time dashboard of all alerts
- Assign staff by zone or expertise
- Escalate emergencies to external services (Fire, Ambulance, Police)
- Broadcast public warnings to all guests with type‑coloured banners
- Download detailed incident logs (`.txt`)
- View analytics dashboard (response times, statistics)
- Two‑way chat with staff members

### 🛡️ Staff
- Receive assigned alerts instantly
- Accept, set severity (Low / Medium / Critical)
- Resolve alerts and add resolution time
- Request escalation if needed
- Chat directly with Emergency Control

### 🤖 AI (Gemini)
- **AI Assistant** for guests → generates personalised safety instructions
- Uses Google Gemini API (model: `gemini‑2.5‑flash`)
- Graceful fallback if API is unavailable

### ☁️ Google Cloud & Real‑time Sync
- **Firebase Firestore** – all alerts and chat messages stored in the cloud
- **Real‑time cross‑device sync** – open on multiple devices and see instant updates
- **Firebase Hosting** – deployed globally with a single command
- **localStorage fallback** – works offline when Firebase is unreachable

### 📊 Analytics & Visualisation
- Live pie chart and bar chart on a separate dashboard
- Response time metrics per emergency type
- Colour‑coded global warning banner based on emergency type

### 🔒 PIN Authentication
- Guest role → no PIN required
- Staff → PIN `1234`
- Emergency Control → PIN `5678`

## 🔧 How to Run Locally

1. **Clone or download** this repository.
2. Open `index.html` in your browser.
3. The app works entirely offline (using localStorage).  
   To enable cloud sync, you need a Firebase project.

### Enable Firebase (real‑time sync)

1. Create a project at [Firebase Console](https://console.firebase.google.com).
2. Enable **Firestore Database** (in production mode).
3. Replace the `firebaseConfig` object in `index.html` with your own config.
4. Set Firestore rules to allow read/write (for demo):
   ```
   rules_version = '2';
   service cloud.firestore {
     match /databases/{database}/documents {
       match /{document=**} {
         allow read, write: if true;
       }
     }
   }
   ```
5. (Optional) Enable **Firebase Hosting** and deploy:
   ```bash
   npm install -g firebase-tools
   firebase login
   firebase init hosting
   firebase deploy --only hosting
   ```

### Enable Gemini AI

1. Get a free API key from [Google AI Studio](https://aistudio.google.com/apikey).
2. Replace the `apiKey` variable in `toggleAIAssistant()` (inside `index.html`).
3. The AI Assistant will appear for guests when they have an active alert.

## 👥 Usage

### Roles & PINs
| Role | PIN (if prompted) |
|---|---|
| Guest | – (no PIN) |
| Emergency Control | `5678` |
| Staff | `1234` |

You can change the PINs in the `switchRole` function.

### Typical Workflow
1. **Guest** reports a fire in Room 205.
2. Alert appears in **EC** panel → staff `S1` (Floor) auto‑assigned.
3. **EC** can manually assign additional staff.
4. **Staff** sees the alert, accepts it, sets severity, and resolves it.
5. System chat logs every action automatically.
6. If the fire is escalated, a red warning banner appears for all guests.
7. EC can trigger a full‑screen evacuation overlay (synced across devices).

## 🧠 Why this project?

During a hotel emergency (fire, medical, security threat and natural disaster), **seconds matter**.  
This system enables:

- Instant communication between guests, staff, and emergency control
- AI‑generated safety instructions for panicking guests
- Real‑time cloud sync so no incident is ever local‑only
- Full audit trails for post‑incident review

Built as a **Google Solution Challenge** hackathon project.
